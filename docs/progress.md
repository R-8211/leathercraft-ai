### 技術的な判断

- **TailwindCSS v4 の選択**: `@tailwindcss/vite` プラグイン方式を採用（v4 では `tailwind.config.js` 不要）。`@import "tailwindcss"` を index.css に記述する形式
- **Noto Serif JP を Google Fonts から読み込み**: 日本語セリフフォントを外部CDNで取得し、初回ロードへの影響を最小化（`display=swap`）
- **プレースホルダー SVG の自作**: hero.png を使わず、実際の型紙に近いSVG（外形・縫い代・折り目・ステッチ穴・寸法注記）を Sprint 1 時点で実装。ユーザーへの価値提案を視覚的に伝える
- **ハッシュアンカーのスムーズスクロール**: 「使い方」リンクはページ内スクロールのため、React Router ではなく `scrollIntoView` で実装
- **App.css は残置**: 既存ファイルは参照されていないが、削除は行わず（仕様外のリファクタリング回避）

### 既知の課題

- Google Fonts が読み込まれないネットワーク環境では、フォールバックフォントになる（機能的問題はなし）
- サンプル型紙 SVG は固定データ。Sprint 4-5 でAI生成SVGに置き換える
- スマホ幅での `details`（FAQ アコーディオン）の `summary` の矢印回転アニメーションは `group-open` の Tailwind v4 サポート状況に依存

**ステータス:** 実装完了 - 評価待ち

### 実装内容

- **SelectPage 本実装** (`frontend/src/pages/SelectPage.jsx`, `SelectPage.module.css`):
  - 7アイテム（二つ折り財布・三つ折り財布・長財布・カードケース・小銭入れ・コインケース・キーケース）のカードグリッド
  - 各カードにアイコン・難易度バッジ（初心者/中級/上級）・所要時間・説明文を表示
  - カードクリックで `/chat/:itemId` へ遷移し、選択アイテム名を `location.state` で渡す
  - ホバー時のアニメーション（枠線変化・シャドウ・わずかな浮き上がり）
  - キーボードフォーカス対応（`focus-visible` スタイル）
  - 初心者向けヒントメッセージを画面下部に表示

- **ChatPage 新規実装** (`frontend/src/pages/ChatPage.jsx`, `ChatPage.module.css`):
  - 左ペイン（チャット）＋右ペイン（型紙プレビュー）の 2 ペイン構成
  - ヘッダー: アイテム名表示・アイテム選択へ戻るボタン
  - メッセージ送信: テキストエリア入力、Enter キー送信、Shift+Enter で改行
  - ユーザー発言: 右寄せ吹き出し（アースブラウン背景）

  - サジェスト例 3 つ: クリックで入力欄に挿入（初回メッセージ前のみ表示）
  - 型紙プレビューエリア: `PatternSvgPlaceholder` + 「型紙はここに表示されます」オーバーレイ
  - タイムスタンプ表示（各メッセージに送受信時刻）
  - スマホ対応: タブ切り替え（チャット / プレビュー）

- **App.jsx 更新**: `/chat/:itemId` ルートを追加。`ChatLayout`（フッターなし）を定義し、チャット画面がビューポート高さをフル活用できる構成に

| 既存機能との統合 | 5 | Sprint 1 の全ページ・コンポーネントが引き続き正常動作。ビルドエラーなし |

### 技術的な判断

- **スタブ AI 応答**: バックエンド未実装のため、キーワードベースのダミー関数 `generateStubResponse()` で実装。Sprint 3 で本物の API 呼び出しに置き換える
- **location.state でアイテム名を受け渡し**: React Router の `useLocation` フックを利用。URL パラメータ（`itemId`）から `ITEM_NAMES` マップで表示名を逆引きするフォールバックも実装
- **ChatLayout（フッターなし）**: チャット画面は `height: calc(100vh - 64px)` でビューポートをフル活用するため、フッターを表示しない専用レイアウトを採用
- **サジェスト例の表示条件**: `messages.length === 1`（初期 AI メッセージのみの状態）かつ `!isLoading` のときのみ表示し、最初のメッセージ送信後は非表示にする
- **PatternSvgPlaceholder の再利用**: 型紙プレビュー右ペインに Sprint 1 で作成した SVG を流用し、オーバーレイで「準備中」状態を表現
- **三つ折り財布を追加**: Sprint 1 の SelectPage プレースホルダーでは 6 アイテムだったが、仕様書に「三つ折り財布」が明記されているため 7 アイテムに変更

### 既知の課題

- スタブ応答のパターンが少なく、想定外の入力では汎用メッセージを返す（Sprint 3 で Claude API に置き換え）
- 型紙プレビュー右ペインはダミー表示のみ（Sprint 4 で SVG 生成後に実データを表示）
- テキストエリアの高さ自動伸長（auto-grow）は未実装（`max-height: 120px` でスクロールで対応）

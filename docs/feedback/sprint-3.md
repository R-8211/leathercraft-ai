## 改善提案

### 優先度：中

1. **追質問の上限強制**: システムプロンプトに「ユーザーから最初の2往復でパラメータが揃わない場合、3回目の応答では必ずデフォルト値で補完して `<requirements>` を出力すること」と明示し、カウンタをシステムプロンプトに含める（APIリクエスト時に往復数を渡す）。

2. **デフォルト値の安全性**: Claude がデフォルト補完する値がスキーマの `min` / `max` の範囲内になるよう、プロンプトに「補完値はスキーマの推奨デフォルト値（下記）を使うこと」と明示し、デフォルト値一覧をプロンプトに含める。現状は Claude が独自の「標準的」値を判断するため、スキーマ外の値が補完される場合がある。

### 優先度：低

3. **要件カードの数値フィールドにも `min`/`max` の UI 表示**: `input[type="number"]` の `min`/`max` 属性は設定されているが、ユーザーには範囲が見えない。ツールチップや placeholder で「70〜130mm」などを示すとユーザビリティが向上する。

4. **Sprint 4 向けプレビューオーバーレイ**: 生成ボタン押下後のプレビューエリアに「Sprint 4 で実装予定」と表示されている。これは仕様に沿った暫定表示であり問題ないが、Sprint 4 では確実に差し替えること。
